# Plan 10 — Build + 200-line + a11y + Lighthouse

## Goal

Verify the project meets every acceptance criterion in `spec.md §11` and
`docs/plans/00-overview.md §Acceptance for the whole effort` before merging
to `main`.

## Inputs

- All earlier plans complete.
- A clean working tree.

## Steps

### 1 · Static budget check (200-line ceiling)

```bash
# every source file under src/ + the Vite shell
find src public/.gitignore index.html vite.config.ts package.json \
     docs/spec.md docs/design.md docs/content-da.md docs/content-en.md \
     docs/plans -type f \
     \( -name '*.ts' -o -name '*.tsx' -o -name '*.css' \
        -o -name '*.html' -o -name '*.md' \) -print0 \
  | xargs -0 awk '
      FNR == 1 { name = FILENAME }
      END {
        if (FNR > 200) {
          printf "OVER  %4d  %s\n", FNR, name
          ec = 1
        }
      }'
```

Any file over 200 lines is a hard fail; split it before merging.

### 2 · No-gradient check

```bash
grep -RInE 'linear-gradient|radial-gradient|conic-gradient' src/styles \
  && { echo "Gradient found — fix before merging."; exit 1; } || true
```

### 3 · Type check + build

```bash
pnpm tsc --noEmit
pnpm build
```

`pnpm build` (with `vite-react-ssg`) must produce:

- `dist/index.html` with the Danish `<title>`, `<meta description>`, and
  `<html lang="da">` baked into the static HTML.
- `dist/en/index.html` with the English equivalents and `<html lang="en">`.
- `dist/assets/` with the JS + CSS bundles.

Quick verifications:

```bash
grep -q 'lang="da"' dist/index.html
grep -q 'lang="en"' dist/en/index.html
grep -q 'Klinik for Manuel Terapi' dist/index.html dist/en/index.html
```

### 4 · Preview

```bash
pnpm preview --port 4173
```

Visit `/` and `/en/` in mobile viewport (375 × 667) and tablet (768 × 1024).
Confirm:

- Hero rating chip + h1 + lead + "Bestil tid" + contact row all visible
  above the fold.
- FAB cluster invisible at top.
- Scroll past hero — FAB cluster fades in.
- Scroll within ~120 px of the footer — FAB cluster fades out.
- Header logo link returns to `#top`.
- DA · EN switcher swaps routes cleanly.

### 5 · Accessibility audit

In Chrome DevTools → Lighthouse, run "Accessibility" + "Best Practices" +
"SEO" + "Performance" with mobile preset on both routes. Goal ≥ 95 in each
of the four categories.

Manual a11y checks (axe DevTools or pa11y CLI):

- Tab through every interactive element; visible focus everywhere.
- Tap-targets ≥ 44 × 44 px.
- Color contrast on `--ink` over the photo overlay ≥ 4.5:1.
- `aria-current="page"` on the active lang switch.
- All anchors with `target="_blank"` also have `rel="noopener"`.

### 6 · Mobile reduced-motion check

In macOS / iOS Settings → Accessibility → Reduce Motion (or DevTools
emulator) — confirm:

- Hero entrance animation is replaced with a fade only.
- FAB cross-fades in/out (no slide).
- No layout jank.

### 7 · Cross-browser smoke test

- iOS Safari (real device, not just emulator) — confirm `position: fixed`
  parallax background does not jitter on rubber-band scroll.
- Firefox + Chrome desktop — check `backdrop-filter` on the sticky header.

### 8 · CI hook (optional, only if a deploy target is set)

If the user has confirmed a deploy target (Vercel / Netlify / GitHub Pages),
add a single workflow that runs steps 1, 2, 3 on push and blocks merge on
failure. Otherwise leave CI for a future plan.

## SSG deferral (2026-05-03)

`vite-react-ssg` integration was attempted and reverted in this pass.
The package's published peer range tops out at `vite ^7` and
`react-router-dom ^6`, while this project ships on Vite 8 + React Router
v7 (which has its own built-in SSG support, per the upstream README).
The build failed at prerender time on
`Package subpath './server.js' is not defined by "exports" in
react-router-dom/package.json` — `vite-react-ssg` imports the v6-only
`react-router-dom/server.js` path that v7 no longer exposes. Forcing it
to work would mean either downgrading react-router-dom to v6 or
overriding peer resolutions, both outside the scope of this plan.

Runtime `<title>` / `<meta description>` / `<html lang>` / hreflang
updates in `src/App.tsx`'s `useEffect` plus the inline
`cp dist/index.html dist/404.html` step in `.github/workflows/deploy-pages.yml`
remain in place. A real SSG/SEO pass — likely via React Router v7's own
`ssr: true` build, or a switch to a Vite 8 + React 19 compatible SSG —
should be a future plan, not a blocking one.
